# Security Policy

## Supported Versions

Currently, the `main` branch and the latest tagged release are supported with security updates. We do not backport security fixes to older unmaintained versions.

| Version | Supported          |
| ------- | ------------------ |
| v1.0.x  | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

We take the security of DocScout-MCP and the Model Context Protocol seriously. This server deals with accessing internal corporate documentation, making security a top priority.

**Do NOT report security vulnerabilities via public GitHub issues.**

If you discover a security vulnerability (such as a flaw in the path traversal mitigation model or API token leakage), please report it by sending an email or via GitHub Private Vulnerability Reporting (if enabled on the repository). 

### What to include in your report
- A description of the vulnerability and its impact
- Steps to reproduce the issue
- Any proposed fixes or mitigation strategies

### Response time
We aim to acknowledge receipt of vulnerability reports within 48 hours and will provide an estimated timeline for a fix. You will be kept updated as we work on a resolution.
