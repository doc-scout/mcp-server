# Release 1.0.0 Preparation — Design Spec

**Date:** 2026-04-16
**Status:** Approved

---

## Overview

DocScout-MCP is feature-complete for a 1.0.0 release. All 29 test suites pass. This spec covers the three areas needed before tagging:

1. **Version injection** — remove the hardcoded `const serverVersion` and inject at build time via ldflags
2. **Documentation corrections** — fix stale tool counts and placeholder strings
3. **CI/CD automation** — add release-drafter, PR labeler, and a release publishing workflow

---

## 1. Version Injection

### Problem

`main.go` declares `const serverVersion = "1.0.0"`. The Dockerfile and Makefile both pass `-X main.version=$(VERSION)` via ldflags, but the Go symbol they target is `main.serverVersion` — the injection is silently a no-op today.

### Fix

**`main.go`** — change from `const` to `var` with a `"dev"` fallback:

```go
var serverVersion = "dev" // set at build time via -ldflags "-X main.serverVersion=<tag>"
```

**`Makefile`** — fix the ldflag symbol name:

```makefile
GOFLAGS := -ldflags="-s -w -X main.serverVersion=$(VERSION)"
```

**`Dockerfile`** — fix the ldflag symbol name in the build step:

```dockerfile
RUN CGO_ENABLED=0 GOOS=$TARGETOS GOARCH=$TARGETARCH \
    go build -ldflags="-s -w -X main.serverVersion=${VERSION}" -o /docscout-mcp .
```

**Result:** Local `go run .` shows `dev`. `make build` shows the git-describe tag. Release binaries and Docker images show the exact `v1.x.x` tag — no manual source edits needed between releases.

## 2. Documentation Corrections

### 2a. README.md — stale tool count

Two places reference `"15 MCP tools"` (the Mermaid diagram and the body paragraph). The actual count is **22**. Both must be updated to `"22 MCP tools"`.

Additionally, two placeholder strings reference `your-username`:

- Claude CLI quick-start snippet: `go run github.com/your-username/docscout-mcp@latest`
- `git clone` URL: `github.com/your-username/docscout-mcp`

Both must use the real path: `github.com/doc-scout/mcp-server`.

### 2b. docs/claude.md — placeholder Docker image

The "Using Docker in Claude Desktop" example references `ghcr.io/your-username/docscout-mcp:latest`. Must be updated to `ghcr.io/doc-scout/mcp-server:latest`.

### 2c. ROADMAP.md and docs/roadmap.md — Semantic Search status

The "Future Work" section lists `### 2. Semantic Search and Vector Embeddings (RAG)` with Phase 2 still open. However, the full `semantic_search` tool with OpenAI/Ollama vector embeddings shipped in PR #23. Phase 2 must be marked complete and moved to the Completed section.

## 3. CI/CD Automation

### 3a. Fix build.yml — remove duplicate lint

The `build` job contains a second `golangci/golangci-lint-action@v7` step (inconsistent version) that duplicates the dedicated `lint` job. Remove it — linting runs once in the `lint` job only.

### 3b. New: .github/release-drafter.yml

Configuration file for the release-drafter action. Defines:

- **Changelog categories:**
  | Label | Section heading |
  |---|---|
  | `breaking` | Breaking Changes |
  | `feature`, `feat` | Features |
  | `bug`, `fix` | Bug Fixes |
  | `documentation`, `docs` | Documentation |
  | `chore`, `deps`, `refactor`, `perf` | Maintenance |

- **Version resolver:** `major` label → major bump; `minor` / `feature` label → minor bump; everything else (default) → patch bump.

- **Tag template:** `v$NEXT_PATCH_VERSION`
- **Name template:** `v$NEXT_PATCH_VERSION`
- **Replacers:** Auto-link issue/PR references.

### 3c. New: .github/labeler.yml

Maps branch name and title commit-prefix patterns to GitHub labels, applied by `actions/labeler` on PR open/sync:

| Pattern                             | Label           |
| ----------------------------------- | --------------- |
| `feat:*` / `feat(*):*`              | `feature`       |
| `fix:*` / `fix(*):*`                | `bug`           |
| `docs:*`                            | `documentation` |
| `chore:*` / `refactor:*` / `perf:*` | `chore`         |
| Title contains `BREAKING CHANGE`    | `breaking`      |

### 3d. New: .github/workflows/release-drafter.yml

Triggers on `push` to `main` and `pull_request` (opened, reopened, synchronize) events.

Jobs:

1. **label** — runs `actions/labeler@v5` to auto-apply labels from `.github/labeler.yml`
2. **draft** — runs `release-drafter/release-drafter@v6` to update the rolling draft release

Permissions: `contents: write`, `pull-requests: write`.

### 3e. New: .github/workflows/release.yml

Triggers on:

- `push: tags: ['v*']`
- `release: types: [published]`

Jobs (run in sequence):

**1. build-binaries**

Matrix: `[{os: linux, arch: amd64}, {os: linux, arch: arm64}, {os: darwin, arch: amd64}, {os: darwin, arch: arm64}, {os: windows, arch: amd64}]`

Steps:

- Cross-compile with `CGO_ENABLED=0 GOOS=$OS GOARCH=$ARCH go build -ldflags="-s -w -X main.serverVersion=${{ github.ref_name }}" -o docscout-mcp-$OS-$ARCH`
- Windows: append `.exe`
- Checksum: `sha256sum` for each binary into `checksums.txt`
- Upload all artifacts

**2. release-assets** (needs: build-binaries)

- Download all artifacts
- Attach binaries + `checksums.txt` to the GitHub Release via `softprops/action-gh-release@v2`

**3. docker** (needs: build-binaries, runs in parallel with release-assets)

- `docker/login-action` to GHCR (`ghcr.io`) using `GITHUB_TOKEN`
- `docker/login-action` to Docker Hub using `DOCKERHUB_USERNAME` + `DOCKERHUB_TOKEN` secrets
- `docker/build-push-action` with:
  - `platforms: linux/amd64,linux/arm64`
  - `build-args: VERSION=${{ github.ref_name }}`
  - Tags: `ghcr.io/doc-scout/mcp-server:${{ github.ref_name }}`, `ghcr.io/doc-scout/mcp-server:latest`, `doc-scout/mcp-server:${{ github.ref_name }}`, `doc-scout/mcp-server:latest`

**Required secrets** (configured in GitHub repo → Settings → Secrets):

- `DOCKERHUB_USERNAME`
- `DOCKERHUB_TOKEN`
- `GITHUB_TOKEN` — provided automatically by Actions

## Success Criteria

- `go test ./...` passes with zero failures
- `go build -ldflags="-X main.serverVersion=v1.0.0" .` produces a binary that reports `v1.0.0`
- Pushing `git tag v1.0.0 && git push --tags` triggers `release.yml`, produces binaries for 5 platform/arch combos, and pushes multi-arch Docker images to both GHCR and Docker Hub
- Merging a PR to `main` updates the release draft with categorized changelog entries
- No `your-username` placeholder appears in any user-facing file
